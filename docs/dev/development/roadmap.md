# 開發藍圖（Development Roadmap）

> 本文件描述 OxOcraft-Manager 的長期發展方向，而非短期待辦事項。

---

# 文件資訊

| 項目   | 內容               |
| ---- | ---------------- |
| 文件版本 | v1.0.0           |
| 狀態   | 持續更新             |
| 適用範圍 | OxOcraft-Manager |
| 最後更新 | YYYY/MM/DD       |

---

# 1. 文件目的

Roadmap 用來描述 OxOcraft-Manager 未來的發展方向。

它不是每日待辦事項（TODO），也不是 Bug 清單，而是記錄專案希望逐步完成的長期目標。

短期開發工作應透過 Issue、TODO 或開發計畫管理，本文件只記錄重要的里程碑。

---

# 2. v0.9.x

**目標：完成核心功能，建立穩定的基礎架構。**

此階段主要著重於功能完善與架構整理，讓 OxOcraft-Manager 具備完整且穩定的核心能力。

主要方向：

* 完成核心功能
* 建立完整文件
* 改善 UI 一致性
* 持續修正 Bug
* 整理程式架構
* 提升整體穩定性

---

# 3. v1.0.x

**目標：完成第一個正式版本。**

當核心功能趨於穩定後，將以正式版為目標，提升整體完成度與使用體驗。

主要方向：

* 提升使用體驗
* 完善說明與教學
* 強化穩定性
* 優化效能
* 持續改善 UI 細節

---

# 4. 未來方向

隨著專案持續發展，OxOcraft-Manager 將依照需求逐步擴充更多管理能力。

可能包含：

* 更多 Minecraft 原生功能支援
* 更完整的伺服器管理工具
* 更完善的玩家管理功能
* 更多資料視覺化與統計分析
* 持續改善操作流程與使用體驗

上述內容僅代表目前規劃方向，不代表未來一定會全部實作。

---

# 5. 不屬於 Roadmap 的內容

以下內容不應記錄於 Roadmap：

* Bug 修正
* 每日開發進度
* 臨時需求
* 尚未確認的想法
* 細部實作流程

這些內容應透過其他開發工具或文件管理。

---

# 6. 更新原則

Roadmap 應隨著專案方向調整，而不是每完成一項功能就修改。

只有當專案的長期規劃發生改變時，才需要更新本文件。

---

# 最終理念

> **Roadmap 描述的是 OxOcraft-Manager 想成為什麼，而不是今天還有哪些事情沒有完成。**
