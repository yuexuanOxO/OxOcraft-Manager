# 設計理念（Design Philosophy）

> 本文件記錄 OxOcraft-Manager 的核心設計理念與重要設計決策。

這些理念不僅影響目前的功能設計，也會作為未來開發、重構與新增功能時的重要依據。

# 1. 幫助玩家理解 Minecraft，而不只是使用 Minecraft

OxOcraft-Manager 最重要的目標，不只是提供一套方便的管理工具。

我更希望玩家能透過使用 OxOcraft-Manager，逐漸理解 Minecraft Server 的運作方式。

因此，許多功能除了提供操作介面外，也會搭配說明、提示與教學，幫助玩家理解每個功能真正的用途，而不是只知道按下哪個按鈕。

理想的結果是，即使有一天不使用 OxOcraft-Manager，玩家仍然能理解 Minecraft Server 的基本管理觀念。

---

# 2. 降低學習門檻，而不是降低功能

方便，不代表隱藏所有細節。

我希望降低的是學習成本，而不是犧牲 Minecraft 原本提供的功能。

OxOcraft-Manager 會盡可能將複雜的設定轉換成容易理解的介面，同時保留進階玩家需要的操作能力。

---

# 3. 好的 UI，是一眼就知道怎麼用

UI 不應該讓玩家思考「下一步該按哪裡」。

而是透過合理的排版、功能分組與適當的引導，讓玩家自然知道每個功能的位置與用途。

此外，也會避免一開始就將大量設定一次呈現在畫面上，而是依照功能關聯性逐步顯示，降低第一次使用時的負擔。

---

# 4. 優先相容原版，而不是強制改造原版

OxOcraft-Manager 以原版 Minecraft Server 為主要支援對象。

設計上會盡量避免修改 `server.jar` 或要求使用者使用特製核心，而是讓玩家可以直接使用官方提供的 Minecraft Server 建立自己的伺服器。

目前部分功能透過 Log、RCON、JSON 或 Datapack 取得資料，這些方式雖然不一定是最理想的技術方案，但能在維持原版相容性的前提下提供足夠的管理能力。

若未來有更穩定且更合適的方式，OxOcraft-Manager 也會逐步調整實作，而不是因為目前的做法可以運作就停止改善。

---

# 5. 必要時介入，但保持透明

部分功能可能會對 Minecraft Server 進行額外的輔助，例如自動備份前的公告、自動解除到期黑名單等。

這些功能的目的，是提升管理便利性，而不是改變 Minecraft 的核心玩法。

因此，OxOcraft-Manager 會盡量讓使用者知道哪些行為來自 OxOcraft-Manager，哪些則是 Minecraft 原本的機制，避免玩家對兩者產生混淆。

---

# 6. 每一個設計，都應該有存在的理由

無論是 UI、程式架構或功能設計，我都希望保持足夠的簡潔。

不為了追求更多功能而增加不必要的複雜度，也不為了追求更大的架構而過度設計。

只有當新的設計能真正改善使用體驗、降低維護成本或提升專案品質時，才值得加入 OxOcraft-Manager。

---

# 最終理念

> **希望 OxOcraft-Manager 不只是管理 Minecraft Server 的工具，而是陪伴玩家理解 Minecraft、學習 Minecraft，並更輕鬆享受 Minecraft 的工具。**
